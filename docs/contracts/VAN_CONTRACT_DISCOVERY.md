VAN_CONTRACT_DISCOVERY
1.0 Overview
This contract defines the mandatory JSON structure for all data transfer between a Scout (Data Probe) and the ScoutCore (Service Layer). Any module producing data for Vanguard must adhere to this schema to ensure system stability and successful state persistence.
2.0 Implementation Notes
Deduplication: The vanguard_id must be generated by the Scout before transmission to allow for immediate rejection of duplicate Entries.
ID Generation: Scouts must sanitize the source_url by removing all session/tracking tokens before executing the SHA-256 hash to prevent database corruption.
Null Handling: If a non-required field cannot be retrieved, the Scout must return null rather than omitting the key.
